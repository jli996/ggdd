## Example: scoring walkthrough

**Query**: `"MMO crafting tier system"`

Semantic similarity scoring (existing):
- Top embedding match might be `coalition-clan-endgame` (MMO content) at ~0.45
- `tier-progression-value-curves` only matches via "tier" → semantic similarity ~0.35

Embedding-based tag boost (new):
- Query embedding compared to each tag embedding:
  - vs `tier-progression` → ~0.62 (strong)
  - vs `mmo` → not a tag anymore (folded into `mmorts`)
  - vs `mmorts` → ~0.50
  - vs `economy` → ~0.35
- For each guide, take MAX over its tags:
  - `tier-progression-value-curves` tags: [puzzle, casual, merge-2, tier-progression, exponential-scaling, economy] → max is `tier-progression` at 0.62 → boost = 0.62 × 0.15 = +0.093
  - `coalition-clan-endgame` tags: [strategy, mmorts, coop, progression] → max is `mmorts` at 0.50 → boost = +0.075
  - Final scores: tier-progression-value-curves = 0.35 + 0.093 = **0.443**; coalition-clan-endgame = 0.45 + 0.075 = 0.525

So in this case `coalition-clan-endgame` still wins (because it has stronger semantic match on the whole phrase), but `tier-progression-value-curves` is now competitive — agents will see BOTH as relevant top results, which is the right answer for "MMO crafting tier system."

## Open questions for review

1. **Per-guide tag distribution** — most guides land at 4-5 tags. A few hit 6 (the casual-puzzle ones that pick up TWO umbrellas: `casual` + `puzzle`). Acceptable?
2. **`casual` umbrella overlap** — 21 guides tagged `casual` (heavy). Is that informative or noise? Could split into `casual-puzzle` + `casual-action` + `casual-economy` sub-umbrellas, but that adds complexity.
3. **`action` umbrella scope** — currently covers action-design + soulslike (6 guides). Should AI-perception also be tagged `action` (since stealth + horror are action-genre-adjacent)? Currently no.
4. **`tag_boost_weight = 0.15` default** — sane starting point or want lower (0.10) / higher (0.20)?
